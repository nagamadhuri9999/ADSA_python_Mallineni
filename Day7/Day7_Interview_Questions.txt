Q1: What is the worst-case time complexity of Linear Search?
A: O(N) because we might have to check every element.

Q2: What is the best-case scenario for Linear Search?
A: O(1), if the target is the very first element.

Q3: When is Linear Search preferred over Binary Search?
A: When the data is unsorted or very small.

Q4: Does Linear Search require additional memory space?
A: No, space complexity is O(1).

Q5: How do you implement Linear Search in Python?
A: Using a standard `for` loop to iterate and check each item.

Q6: What happens if the target is not found in Linear Search?
A: It returns -1 or None after checking all N elements.

Q7: Is Linear Search efficient for an array of 1 million items?
A: No, it is highly inefficient for large datasets.

Q8: Can Linear Search be used on linked lists?
A: Yes, it is exactly the same concept: traverse node by node.

Q9: Does Python's `in` keyword use Linear Search?
A: Yes, for standard lists, `in` operates in O(N) time.

Q10: What is the average case time complexity of Linear Search?
A: O(N/2), which mathematically simplifies to O(N).

Q26: What is the strict prerequisite for Binary Search?
A: The collection MUST be sorted first.

Q27: What is the worst-case time complexity of Binary Search?
A: O(log N).

Q28: What is the best-case scenario for Binary Search?
A: O(1), if the target is exactly in the middle on the first check.

Q29: How does Binary Search work?
A: It finds the middle element and halves the search space recursively or iteratively.

Q30: Why do we use `low <= high` in the while loop?
A: Because the target might be at the exact index where low and high converge.

Q31: How do you safely calculate the `mid` index?
A: mid = low + (high - low) // 2 to prevent integer overflow in compiled languages.

Q32: What is the space complexity of iterative Binary Search?
A: O(1), as no extra memory is needed.

Q33: What is the space complexity of recursive Binary Search?
A: O(log N) due to the call stack frames.

Q34: Can Binary Search be applied to a Hash Map?
A: No, Hash Maps have no sequence order and rely on hashing instead.

Q35: If an array has 1,000,000 items, how many checks does Binary Search make?
A: Roughly 20 checks, since 2^20 is approx 1,000,000.
