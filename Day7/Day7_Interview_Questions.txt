Day 7: 100 Interview Questions on Space Complexity and Sorting
============================================================

Q1: What does Space Complexity measure?
A: It measures the amount of extra memory an algorithm uses relative to the input size.

Q2: What is the difference between Auxiliary Space and Space Complexity?
A: Auxiliary space is the extra or temporary space used by an algorithm. Space complexity includes both auxiliary space and space used by the input.

Q3: What does O(1) space complexity mean?
A: It means the algorithm uses a constant amount of extra memory, regardless of input size.

Q4: Do in-place sorting algorithms use O(N) space?
A: No, in-place sorting algorithms typically use O(1) auxiliary space.

Q5: What happens to space complexity if we create a copy of an array of size N?
A: The space complexity becomes O(N).

Q6: What is the prerequisite for Binary Search?
A: The array must be sorted.

Q7: What is the worst-case time complexity of Binary Search?
A: O(log N).

Q8: What is the space complexity of iterative Binary Search?
A: O(1).

Q9: What is the space complexity of recursive Binary Search?
A: O(log N) due to the call stack.

Q10: How does Binary Search work?
A: It repeatedly divides the search interval in half.

Q11: Explain the core concept of Bubble Sort.
A: It repeatedly steps through the list, compares adjacent elements and swaps them if they are in the wrong order.

Q12: What is the worst-case time complexity of Bubble Sort?
A: O(N^2).

Q13: What is the best-case time complexity of Bubble Sort?
A: O(N) when the array is already sorted and an early-exit optimization is used.

Q14: What is the space complexity of Bubble Sort?
A: O(1) because it sorts in-place.

Q15: Why is Bubble Sort rarely used in practice?
A: Its O(N^2) average and worst-case time complexity makes it highly inefficient for large datasets.

Q16: Explain the core concept of Selection Sort.
A: It divides the array into sorted and unsorted parts. It repeatedly selects the smallest element from the unsorted part and swaps it with the first unsorted element.

Q17: What is the worst-case time complexity of Selection Sort?
A: O(N^2).

Q18: Does Selection Sort have an O(N) best case like Bubble Sort?
A: No, Selection Sort is always O(N^2) even if the array is sorted, because it still scans the remaining elements to find the minimum.

Q19: What is the space complexity of Selection Sort?
A: O(1) because it sorts in-place.

Q20: When might Selection Sort be preferred over Bubble Sort?
A: When writing to memory is expensive, as Selection Sort makes at most O(N) swaps compared to Bubble Sort's O(N^2) swaps.

Q21: Explain the core concept of Insertion Sort.
A: It builds the final sorted array one element at a time by taking one element and inserting it into its correct position in the already sorted part.

Q22: What is the worst-case time complexity of Insertion Sort?
A: O(N^2).

Q23: What is the best-case time complexity of Insertion Sort?
A: O(N) when the array is already sorted, because the inner loop immediately breaks.

Q24: What is the space complexity of Insertion Sort?
A: O(1) because it sorts in-place.

Q25: Why is Insertion Sort useful in practice?
A: It is highly efficient for small data sets or data that is already substantially sorted.
