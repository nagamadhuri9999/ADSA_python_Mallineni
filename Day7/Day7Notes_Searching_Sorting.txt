# Day 7 Notes: Space Complexity, Searching, and Sorting Algorithms

## 1. Space Complexity
- What is it?: Just like Time Complexity measures how long an algorithm takes based on input size N, Space Complexity measures how much *extra memory* an algorithm requires as N grows.
- Auxiliary Space: The extra space or temporary space used by an algorithm, excluding the space taken by the input itself. 
- O(1) Space: If an algorithm only uses a fixed number of variables (like a few counters or pointers) regardless of how large the input array is, its space complexity is O(1). All three sorting algorithms learned today (Bubble, Selection, Insertion) sort the array "in-place", meaning they have O(1) space complexity.
- O(N) Space: If you create a new array of the same size as the input to help you solve the problem, the space complexity is O(N).

## 2. Binary Search (Review)
- Pre-requisite: The collection MUST be sorted.
- How it works: 
  1. Find the middle element `mid = (low + high) // 2`.
  2. If the target is the middle element, return the index.
  3. If the target is less than the middle element, narrow the search to the left half `high = mid - 1`.
  4. If the target is greater than the middle element, narrow the search to the right half `low = mid + 1`.
- Time Complexity: O(log N). Every step cuts the remaining search space in half.
- Space Complexity: Iterative is O(1). Recursive is O(log N) due to the call stack frames.

## 3. Bubble Sort
- Concept: Compares adjacent elements and swaps them if they are in the wrong order. This process "bubbles" the largest elements to the end of the list.
- Passes: Requires N passes. After the 1st pass, the largest element is at the very end. After the 2nd pass, the 2nd largest is in its correct place, and so on.
- Optimization: If a full pass is made without any swaps, the array is already sorted, and the algorithm can terminate early.
- Time Complexity: O(N^2) in average and worst cases. Best case (with optimization) is O(N) if the array is already sorted.
- Space Complexity: O(1).

## 4. Selection Sort
- Concept: Divides the input into a sorted and an unsorted region. It repeatedly finds the minimum element from the unsorted region and swaps it with the first element of the unsorted region.
- Passes: In each pass, the unsorted region shrinks by one element from the left, and the sorted region grows.
- Time Complexity: O(N^2) in all cases (Best, Average, Worst) because it always scans the remaining unsorted elements to find the minimum.
- Space Complexity: O(1).
- Advantage over Bubble Sort: Makes at most O(N) swaps, whereas Bubble Sort makes O(N^2) swaps. Useful when writing to memory is significantly more expensive than reading.

## 5. Insertion Sort
- Concept: Builds the final sorted array one item at a time. It iterates through the input elements and grows a sorted output list. At each iteration, it removes one element from the input data, finds the location it belongs within the sorted list, and inserts it there.
- Analogy: Like sorting playing cards in your hands. You pick a new card and place it in the correct position among the cards you are already holding.
- Time Complexity: O(N^2) in average and worst cases. Best case is O(N) if the array is already sorted.
- Space Complexity: O(1).
- Practical Application: Very efficient for small arrays or nearly sorted arrays. Often used as the base case for more complex algorithms like Merge Sort or Quick Sort.
