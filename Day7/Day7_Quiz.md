# Day 7 Quiz: Space Complexity & Sorting Algorithms

**1. What is Space Complexity?**
a) The total size of the hard drive required to run a program
b) The amount of time an algorithm takes to run
c) The amount of extra memory an algorithm requires relative to the input size
d) The number of lines of code in a program
<details>
<summary>Answer</summary>
c) The amount of extra memory an algorithm requires relative to the input size
</details>

**2. Which of these sorting algorithms has a best-case time complexity of O(N)?**
a) Bubble Sort (Optimized)
b) Selection Sort
c) Insertion Sort
d) Both A and C
<details>
<summary>Answer</summary>
d) Both A and C. Insertion sort and an optimized Bubble Sort can exit in O(N) if the array is already sorted.
</details>

**3. What is the Space Complexity of Bubble, Selection, and Insertion Sort?**
a) O(N)
b) O(1)
c) O(log N)
d) O(N^2)
<details>
<summary>Answer</summary>
b) O(1). They all sort the array in-place, requiring no extra data structures proportional to input size.
</details>

**4. How does Selection Sort work?**
a) By repeatedly swapping adjacent elements
b) By finding the minimum element and placing it at the beginning
c) By building a sorted portion one element at a time from left to right
d) By dividing the array in half
<details>
<summary>Answer</summary>
b) By finding the minimum element and placing it at the beginning
</details>

**5. Which algorithm is generally considered most efficient for small datasets or nearly sorted datasets?**
a) Bubble Sort
b) Selection Sort
c) Insertion Sort
d) They are all exactly the same
<details>
<summary>Answer</summary>
c) Insertion Sort
</details>
