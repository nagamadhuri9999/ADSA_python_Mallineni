Day 8: Divide and Conquer

Breaking the O(n^2) barrier with Merge Sort and Quick Sort.

1. The Divide and Conquer Paradigm

This is a powerful algorithmic technique where you break a problem into smaller subproblems, solve those subproblems recursively, and then combine the solutions.

2. Visualizing Merge Sort

Merge Sort splits the array in half until it reaches single elements, then merges them back together in sorted order. Time Complexity: O(n log n).

# The crucial merge step
def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] < right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

3. Merge Sort vs Quick Sort

Algorithm Best Time Worst Time Space
Merge SortO(n log n)O(n log n)O(n)
Quick SortO(n log n)O(n^2)O(log n)

Quick Sort is usually faster in practice because it operates in-place, reducing memory overhead, despite its theoretical worst-case.
