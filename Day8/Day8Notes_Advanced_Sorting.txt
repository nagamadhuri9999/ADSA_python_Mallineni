# Day 8 Notes: Advanced Sorting Algorithms & Complexity

## 1. Introduction to Recursion
Recursion is a programming technique where a function calls itself to solve smaller instances of the same problem.
It is the foundation of many advanced algorithms, including Divide and Conquer.

A recursive function must have two essential parts:
1. **Base Case**: The condition under which the function stops calling itself to prevent an infinite loop (Stack Overflow).
2. **Recursive Case**: The part of the function where it calls itself with a modified, smaller input, moving towards the base case.

**Example: Factorial**
`5! = 5 * 4!`, which means `n! = n * (n-1)!`.

---

## 2. The Divide & Conquer Paradigm
Divide and Conquer is a fundamental algorithmic strategy that involves three steps:
1. **Divide**: Break the problem into smaller, non-overlapping subproblems.
2. **Conquer**: Solve the subproblems recursively.
3. **Combine**: Merge the solutions of the subproblems to form the final solution.
Both Merge Sort and Quick Sort utilize this paradigm.

---

## 2. Merge Sort
Merge Sort divides the array into two halves until each sub-array contains only one element (which is naturally sorted). Then, it repeatedly merges the sub-arrays to produce new sorted sub-arrays until there is only one sorted array remaining.

**Key Characteristics:**
- **Time Complexity**: O(N log N) in the Best, Average, and Worst cases. 
- **Space Complexity**: O(N) because it requires a temporary array during the merge phase.
- **Stability**: It is a **stable** sort (equal elements retain their relative order).

**When to use Merge Sort:**
- When stability is required.
- When sorting linked lists (since merging linked lists doesn't require extra O(N) space).
- When worst-case O(N log N) guarantee is needed.

---

## 3. Quick Sort
Quick Sort works by selecting a "pivot" element and partitioning the array around it, such that elements smaller than the pivot are placed before it, and elements greater are placed after it. This process is applied recursively to the sub-arrays.

**Key Characteristics:**
- **Time Complexity**: 
  - Best & Average: O(N log N)
  - Worst: O(N^2) (occurs when the array is already sorted and the pivot is the largest/smallest element).
- **Space Complexity**: O(log N) for the recursive call stack. It sorts "in-place".
- **Stability**: It is an **unstable** sort (equal elements might lose their original relative order).

**When to use Quick Sort:**
- It is the preferred algorithm for sorting arrays in memory due to good cache locality and smaller constant factors.
- When in-place sorting is preferred over O(N) space.

## 5. Stable vs. Unstable Sorting
A sorting algorithm is **stable** if it preserves the relative order of equal elements. 
For example, if sorting a list of students primarily by Grade and secondarily by Name, a stable sort ensures that students with the same Grade remain sorted by Name (if they were already sorted by Name prior).

Stable Sorts: Bubble, Insertion, Merge Sort.
Unstable Sorts: Selection, Quick Sort.
