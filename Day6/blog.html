Day 6: Big O & Searching

Mastering Algorithmic Complexity.

1. Big O Notation

Big O describes the worst-case scenario: how the runtime grows as the input data gets massive.

  • O(1): Constant Time (Instant lookup)
  • O(log n): Logarithmic Time (Binary Search)
  • O(n): Linear Time (Loops)
  • O(n^2): Quadratic Time (Nested Loops)

2. Binary Search Dry Run

Binary Search works by repeatedly halving the search space. The array must be sorted!
