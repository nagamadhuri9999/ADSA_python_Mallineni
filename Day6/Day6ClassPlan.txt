Day 6: Time and Space Complexity - Big O Notation (6-hour online class)

Session goal:
Understand how to measure the performance of algorithms independently of hardware. Learn to classify algorithms using Big O Notation and evaluate code snippets for their Time and Space Complexity.

Class structure:

1. Welcome and Review (30 minutes)
- Recap Day 5: Loops and frequency counting.
- Why do we need Big O? Explain that timing code with a stopwatch depends on the computer's speed, whereas Big O measures the number of operations relative to the input size (N).

2. The Big O Paradigm: O(1) and O(N) (60 minutes)
- O(1) Constant Time: Operations that take the same amount of time regardless of input size (e.g., accessing an array by index, math operations).
- O(N) Linear Time: Operations that grow proportionally with input size (e.g., standard `for` loops).
- Linear Search: Using a simple sequence scan to introduce O(N) searching.
- Dropping Constants: Why O(2N) is simplified to O(N).

3. Nested Loops and O(N^2) (45 minutes)
- Analyzing nested loops.
- Nested loops dry run only: Step-by-step trace of outer and inner loops over small input sizes.
- O(N^2) Quadratic Time: Growth is exponential (e.g., comparing every element to every other element, Bubble Sort).
- Interview-style question: "If an array has 10 elements, an O(N^2) algorithm takes 100 steps. How many steps if the array has 100 elements?" (10,000 steps!).

Break (15 minutes)

4. Introduction to O(log N) Logarithmic Time (45 minutes)
- The concept of halving the search space.
- The phonebook analogy: Finding a name in a dictionary by tearing it in half vs checking every page.
- Teasing Binary Search (which will be fully implemented tomorrow on Day 7).

5. Space Complexity (30 minutes)
- Measuring how much *extra* memory an algorithm needs.
- O(1) Space: Modifying arrays in-place or using a few integer variables.
- O(N) Space: Creating a new list or dictionary that scales with the input size.

6. Code Optimizations: Factors and Prime Numbers (45 minutes)
- Brute Force Approach: Finding factors or checking primes in O(N) time.
- The Square Root Logic: Identifying that factors come in pairs.
- Code Optimizations for prime number program and factors program.
- Optimization to O(sqrt(N)) Time Complexity.

Break (15 minutes)

7. Practice: Identifying Big O (45 minutes)
- Live exercises: Show 10 different code snippets on screen.
- Students must identify both Time and Space complexity.
- Tricky cases: Two separate loops `O(N) + O(N) = O(N)` vs Nested loops `O(N) * O(N) = O(N^2)`.
- Loop with step > 1 (e.g., `for i in range(0, N, 2)`) is still O(N).

8. Review, Q&A, and Homework (30 minutes)
- Review the major complexities: O(1), O(sqrt(N)), O(log N), O(N), O(N^2).
- Answer student questions.
- Assign homework: 100 Big O Practice Snippets and 100 Q&A.
- Prepare for Day 7: Applying this to Searching Algorithms!
