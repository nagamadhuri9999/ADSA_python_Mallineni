Day 6: 100 High-Quality Interview Questions on Big O Notation
============================================================

Q1: What does Big O notation mathematically represent?
A: It represents the asymptotic upper bound (worst-case scenario) of an algorithm's complexity.

Q2: What is the time complexity of appending an element to the end of a Python list?
A: O(1) amortized, because Python lists are dynamic arrays that allocate extra space in advance.

Q3: What is the time complexity of inserting an element at the BEGINNING of a Python list?
A: O(N), because every existing element must be shifted one position to the right in memory.

Q4: What is the time complexity of looking up a key in a Python dictionary?
A: O(1) on average, because dictionaries are implemented as Hash Maps.

Q5: What is the difference between Time Complexity and Space Complexity in the context of Big O Notation?
A: Time complexity measures operations, while space complexity measures the extra memory allocated relative to input size.

Q6: How does Big O Notation behave for a very small input size?
A: For small inputs, even inefficient algorithms (like O(N^2)) can run fast. Big O only matters as N approaches infinity.

Q7: If an algorithm has multiple phases, how do you determine its final Big O Notation?
A: You drop the non-dominant terms. For example, O(N^2) + O(N) simplifies to O(N^2).

Q8: What is the worst-case scenario for Big O Notation?
A: Big O generally represents the upper bound, meaning the maximum number of operations an algorithm could possibly execute.

Q9: Can you optimize an algorithm to improve its Big O Notation?
A: Yes, often by trading space for time (e.g., using Hash Maps) or applying divide-and-conquer strategies.

Q10: How do constant multipliers affect Big O Notation?
A: They don't. O(5N) simplifies to O(N) because constants are ignored in asymptotic analysis.

Q11: What does O(1) Constant Time measure?
A: It measures how the runtime or space requirements of an algorithm grow as the input size (N) grows towards infinity.

Q12: Give an example of an O(1) Constant Time operation.
A: Depends on the class: O(1) is array lookup, O(N) is single loop, O(N^2) is nested loop.

Q13: Why is O(1) Constant Time important in software engineering?
A: It allows developers to predict scalability and prevent performance bottlenecks before they occur in production.

Q14: Does O(1) Constant Time care about hardware execution speed?
A: No, Big O ignores hardware differences and constant factors, focusing purely on algorithmic growth rates.
